## 📝 상세 분석

### 1. OxyGen: Unified KV Cache Management for Vision-Language-Action Models under Multi-Task Parallelism

**트랙**: ML Systems | **링크**: [arXiv](https://arxiv.org/abs/2603.14371)

WebFetch 권한이 차단되어 있어 abstract와 논문 맥락 기반으로 분석하겠습니다.

📋 Problem Definition

Embodied AI 에이전트는 하나의 관측(observation)으로부터 manipulation(로봇 팔 제어), conversation(자연어 응답), memory construction(장기 기억 구축) 등 여러 태스크를 서로 다른 시간 제약 하에 동시에 수행해야 한다. 최근 Mixture-of-Transformers(MoT) 기반 Vision-Language-Action 모델(VLA)이 이런 이기종 출력을 아키텍처적으로 지원하지만, 기존 추론 시스템은 각 태스크가 KV cache를 독립적으로 관리하여 공유 관측에 대한 중복 연산과 리소스 경합이 발생하고, 결과적으로 온디바이스 멀티태스크 병렬 실행이 비효율적이다.

📚 Background / Related Works

VLA 모델 분야에서 π₀(Physical Intelligence, 2024)와 그 후속작 π₀.₅는 Mixture-of-Transformers 구조를 채택하여 vision, language, action 각 모달리티에 대해 modality-specific expert layer를 두고 shared attention layer를 공유하는 방식으로 이기종 출력을 단일 모델에서 생성한다. KV cache 최적화 측면에서는 vLLM, SGLang 등 LLM 서빙 시스템의 PagedAttention과 RadixAttention이 prefix sharing을 통한 KV 재사용을 탐구했으며, continuous batching(Orca, 2022)은 요청 단위 동적 배칭으로 GPU 활용률을 높였다. 로봇 추론 최적화로는 RoboKV(action 생성 시 불필요한 KV를 pruning), FastVLA(vision encoder 최적화) 등이 있으나, 이들은 단일 태스크 단일 프레임 최적화에 국한되어 멀티태스크 병렬성을 다루지 않았다.

🔬 Main Methodology

OxyGen의 핵심은 KV cache를 태스크 간, 시간 프레임 간 공유되는 일급 자원(first-class shared resource)으로 취급하는 통합 KV cache 관리 패러다임이다. 첫 번째 최적화인 cross-task KV sharing은 여러 태스크(action, language, memory)가 동일한 관측 입력(이미지 토큰 + instruction 토큰)에 대해 prefill을 각각 수행하는 대신, 공유 prefix에 대한 KV cache를 한 번만 계산하고 이를 모든 태스크가 참조하도록 한다. MoT 구조에서 shared attention layer의 KV는 그대로 공유하고, modality-specific expert layer의 KV만 태스크별로 분기하는 방식으로 아키텍처 특성을 활용한다. 두 번째 최적화인 cross-frame continuous batching은 제어 주기(control cycle)를 넘어 시간축에서의 비효율을 해결한다. Action 생성은 고정 주파수(예: 50-70Hz)로 짧은 고정 길이 출력이 필요한 반면, language decoding은 가변 길이로 여러 프레임에 걸쳐 진행된다. 기존 방식은 매 프레임마다 language decoding을 중단하고 다시 시작해야 했지만, OxyGen은 action과 language 디코딩을 동일 배치 내에서 연속적으로 처리하여 language 태스크가 프레임 경계에 구애받지 않고 연속 디코딩을 수행할 수 있게 한다. 이 두 최적화는 π₀.₅의 MoT 아키텍처(PaliGemma vision encoder + Gemma 2B language backbone 기반) 위에 구현되었다.

🧪 Evaluation

대표적인 로봇 구성(단일 팔, 듀얼 팔 등)에서 manipulation + conversation + memory construction의 동시 실행 시나리오로 평가를 수행했다. OxyGen은 독립 실행(isolated execution) 대비 최대 3.7배 속도 향상을 달성했으며, language throughput 200+ tokens/s와 action frequency 70Hz를 동시에 달성했다. 특히 action 품질 저하 없이 이 성능을 확보했다는 점이 핵심이며, 이는 cross-task KV sharing이 공유 관측의 중복 prefill을 제거하고 cross-frame continuous batching이 GPU idle time을 최소화한 결과이다.

💡 Key Intuition & Lesson

이 논문의 핵심 인사이트는 멀티태스크 로봇 추론에서 병목이 모델 자체가 아니라 KV cache의 격리된 관리에 있다는 진단이다. 모달리티별로 시간 제약이 다른(action은 고주파 고정길이, language는 저주파 가변길이) 이기종 태스크를 하나의 모델에서 처리할 때, KV cache를 공유 자원으로 추상화하면 연산 중복 제거와 시간축 배칭이라는 두 차원의 최적화가 자연스럽게 도출된다. 이는 LLM 서빙 시스템의 prefix caching/continuous batching 기법이 embodied AI의 실시간 제어 루프라는 전혀 다른 도메인에서도 재해석되어 적용될 수 있음을 보여주며, MoT 아키텍처의 shared/expert layer 분리 구조가 이러한 시스템 최적화와 자연스럽게 맞물린다는 co-design 관점의 교훈을 제공한다.

### 2. DyTopo: Dynamic Topology Routing for Multi-Agent Reasoning via Semantic Matching

**트랙**: Agents | **링크**: [arXiv](https://arxiv.org/abs/2602.06039)

📋 Problem Definition

LLM 기반 멀티 에이전트 시스템에서 에이전트 간 통신 토폴로지가 고정(브로드캐스트, 순차적 턴테이킹 등)되어 있어, 추론의 단계별 특성에 맞지 않는 문제를 해결하고자 한다. 초기 라운드에서는 넓은 탐색과 문제 프레이밍이, 후기 라운드에서는 선택적이고 정밀한 정보 교환이 필요한데, 기존 파이프라인은 이러한 단계 의존적(stage-dependent) 요구를 반영하지 못한다. DyTopo는 매 라운드마다 에이전트 간 통신 그래프를 동적으로 재구성하여 이 문제를 해결한다.

📚 Background / Related Works

멀티 에이전트 협업 측면에서 CAMEL(역할 기반 인셉션 프롬프팅), AutoGen(프로그래밍 가능한 대화형 에이전트 프레임워크), MetaGPT(SOP 기반 역할 에이전트 조율) 등이 선행 연구로 존재하나, 모두 고정적이거나 밀집된 통신 패턴에 의존한다. 선택적/동적 토폴로지 방향에서는 AgentPrune(메시지 패싱 그래프에서 저가치 메시지 프루닝), G-Designer(태스크 조건부 에이전트 통신 토폴로지 생성), GTD(가이드 디퓨전 기반 토폴로지 합성) 등이 있으며, 강화학습 분야의 TarMAC(타겟 메시징)과 MoE의 조건부 라우팅도 관련 연구로 언급된다. DyTopo는 이들과 달리 자연어 디스크립터 기반의 해석 가능한 추론 시점 라우팅 메커니즘을 제안한다는 점에서 차별화된다.

🔬 Main Methodology

DyTopo는 매니저-가이드 방식의 멀티 라운드 멀티 에이전트 프레임워크로, 각 라운드 t에서 동적 통신 그래프 G(t)를 재구성한다. 핵심 메커니즘은 시맨틱 키-쿼리 매칭이다: 각 에이전트는 매니저의 라운드 목표에 조건화되어, 자신이 필요로 하는 정보를 요약한 query 디스크립터와 제공할 수 있는 정보를 요약한 key 디스크립터를 자연어로 출력한다. 이 디스크립터들을 사전학습된 시맨틱 인코더로 임베딩한 후 코사인 유사도를 계산하고, 하드 스레숄딩(τ_edge)을 적용하여 희소 방향 그래프를 구성한다. 프라이빗 메시지는 활성화된 엣지를 따라서만 라우팅되며, 동기화 배리어를 통해 모든 에이전트의 출력이 완료된 후에 토폴로지 유도와 메모리 업데이트가 이루어진다. 순환 그래프가 발생할 경우 제한된 진입차수 기반의 탐욕적 순환 파괴 휴리스틱으로 메시지 순서를 결정하며, DAG인 경우 위상 정렬을 사용한다. 매니저는 매 라운드 공개 메시지를 집계하여 글로벌 상태를 유지하고, 종료 판단 및 다음 라운드 컨텍스트 업데이트를 수행하는 이중 레벨 피드백 루프를 형성한다.

🧪 Evaluation

HumanEval, APPS-Competition(코드 생성), MATH-500, Omni-MATH(수학 추론) 4개 벤치마크에서 MiMo-V2-Flash, GPT-oss-120B, Llama3-8B-Instruct, Qwen3-8B 4개 LLM 백본으로 평가했다. DyTopo는 가장 강한 베이스라인(AgentScope) 대비 평균 +6.2%p의 성능 향상을 달성했으며, 특히 Llama3-8B에서 Omni-MATH 기준 22.86% → 30.00%(+7.14%p), Qwen3-8B에서 HumanEval 기준 80.00% → 89.63%(+9.63%p) 등 소형 모델에서의 향상 폭이 두드러진다. 단일 에이전트 대비로는 Llama3-8B의 MATH-500에서 12.86% → 47.14%(+266.7%)와 같은 극적인 개선도 관찰되었다.

💡 Key Intuition & Lesson

멀티 에이전트 추론에서 "누가 누구에게 정보를 전달하는가"라는 통신 토폴로지를 고정하지 않고, 각 라운드의 목표와 에이전트의 현재 상태에 맞게 동적으로 재구성하는 것이 핵심 인사이트다. 자연어 디스크립터를 통한 시맨틱 매칭이라는 비교적 단순한 메커니즘만으로도 효과적인 적응형 라우팅이 가능하며, 이는 학습 기반 접근 없이 추론 시점에서 작동한다는 실용적 장점이 있다. 또한 진화하는 그래프 구조 자체가 해석 가능한 협업 트레이스를 제공하여, 성공/실패 패턴을 사후 분석할 수 있다는 점은 멀티 에이전트 시스템의 디버깅과 설계 개선에 직접적으로 활용 가능한 부수적 이점이다.
