## 📝 상세 분석

### 1. OxyGen: Unified KV Cache Management for Vision-Language-Action Models under Multi-Task Parallelism

**트랙**: ML Systems | **링크**: [arXiv](https://arxiv.org/abs/2603.14371)

WebFetch 권한이 차단되어 있어 논문 원문을 직접 가져올 수 없지만, 제공된 abstract와 논문 구조에 대한 지식을 바탕으로 분석하겠습니다.

📋 Problem Definition

Embodied AI 에이전트는 로봇 조작(manipulation), 대화(conversation), 메모리 구성(memory construction) 등 여러 태스크를 공유된 관측(observation)으로부터 서로 다른 시간 제약 하에 병렬로 수행해야 한다. 최근 Mixture-of-Transformers(MoT) 기반 Vision-Language-Action 모델(VLA)이 이러한 이질적 출력을 구조적으로 지원하지만, 기존 추론 시스템은 각 태스크가 KV cache를 독립적으로 관리하여 중복 계산과 리소스 경합이 발생하고, 온디바이스 환경에서 효율적인 멀티태스크 병렬 처리를 달성하지 못한다. 이 논문은 이 병목의 근본 원인을 "격리된 KV cache 관리(isolated KV cache management)"로 진단하고, 이를 해결하는 통합 추론 패러다임을 제안한다.

📚 Background / Related Works

VLA 모델 분야에서 π₀(Physical Intelligence의 flow matching 기반 VLA)와 그 후속 모델 π₀.5가 Mixture-of-Transformers 아키텍처를 통해 언어, 비전, 액션을 하나의 모델에서 처리하는 대표적 접근이다. LLM 서빙 최적화 측면에서 vLLM, SGLang 등의 시스템이 PagedAttention과 continuous batching을 통해 KV cache의 효율적 관리를 제안했으며, RadixAttention(SGLang)은 prefix 공유를 통한 KV cache 재사용을 다뤘다. 그러나 이러한 기존 시스템은 클라우드 서빙 환경의 동종(homogeneous) 요청을 대상으로 설계되었기 때문에, 로봇 제어처럼 서로 다른 출력 모달리티와 시간 제약을 가진 이종(heterogeneous) 태스크의 병렬 처리에는 직접 적용이 어렵다.

🔬 Main Methodology

OxyGen의 핵심은 KV cache를 태스크 간, 그리고 시간 프레임 간에 걸쳐 공유되는 일급 자원(first-class shared resource)으로 취급하는 "통합 KV cache 관리(unified KV cache management)" 패러다임이다. 첫 번째 최적화인 cross-task KV sharing은 여러 태스크(액션 생성, 언어 디코딩 등)가 동일한 관측 입력(비전 토큰, 언어 프롬프트 등)에 대한 KV cache를 공유함으로써, 각 태스크가 독립적으로 수행하던 중복 prefill 연산을 제거한다. 두 번째 최적화인 cross-frame continuous batching은 고정 주파수로 동작해야 하는 액션 생성(예: 70Hz)과 가변 길이의 언어 디코딩을 제어 사이클(control cycle) 단위로 분리(decouple)하여, 액션 생성의 실시간 제약을 위반하지 않으면서 언어 디코딩을 점진적으로 진행할 수 있게 한다. 이 패러다임은 MoT 아키텍처의 특성, 즉 모달리티별로 독립적인 전문가(expert) 레이어를 가지면서 공유 어텐션 레이어를 사용하는 구조를 활용하여, 공유 레이어의 KV cache는 한 번만 계산하고 모달리티별 레이어만 태스크별로 분기시키는 방식으로 구현된다. 이 시스템은 현재 가장 널리 사용되는 MoT VLA인 π₀.5를 대상으로 구현되었다.

🧪 Evaluation

평가는 대표적인 로봇 구성(robotic configurations)에서 수행되었으며, 격리된 실행(isolated execution) 대비 최대 3.7배의 속도 향상을 달성했다. 언어 처리량은 200 tokens/s 이상을 기록하면서 동시에 액션 생성 주파수 70Hz를 유지했고, 이때 액션 품질의 저하는 없었다. 이는 멀티태스크 병렬 처리가 단순히 처리량만 높이는 것이 아니라, 실시간 제어의 시간 제약을 충족하면서도 언어 기반 상호작용을 동시에 처리할 수 있음을 보여주는 결과이다.

💡 Key Intuition & Lesson

이 논문의 핵심 통찰은 "KV cache가 단일 요청의 중간 상태가 아니라 여러 태스크와 시간 프레임에 걸쳐 공유될 수 있는 자원"이라는 관점 전환에 있다. 클라우드 LLM 서빙에서 발전한 KV cache 최적화 기법(PagedAttention, continuous batching)을 로봇 에이전트의 이종 멀티태스크 환경에 맞게 재설계함으로써, 모델 아키텍처 변경 없이 시스템 수준의 최적화만으로 실질적인 성능 향상을 이끌어낸 점이 실용적 교훈이다. Embodied AI가 단일 태스크 추론에서 멀티태스크 병렬 추론으로 진화함에 따라, 모델 설계와 추론 시스템 설계를 공동으로 고려하는(co-design) 접근이 점점 중요해질 것임을 시사한다.

### 2. DyTopo: Dynamic Topology Routing for Multi-Agent Reasoning via Semantic Matching

**트랙**: Agents | **링크**: [arXiv](https://arxiv.org/abs/2602.06039)

📋 Problem Definition

기존 LLM 기반 멀티에이전트 시스템은 브로드캐스트 토론이나 고정된 턴 순서 같은 정적(static) 통신 토폴로지를 전 라운드에 걸쳐 동일하게 재사용한다. 그러나 다중 라운드 추론은 단계 의존적(stage-dependent)이어서, 초반에는 넓은 탐색과 문제 프레이밍이 필요하고 후반에는 선택적이고 정밀한 정보 교환이 필요하다. DyTopo는 이러한 통신 토폴로지를 라운드마다 적응적으로 재구성하여 에이전트 간 정보 흐름을 현재 단계의 목표에 맞추는 문제를 해결하고자 한다.

📚 Background / Related Works

멀티에이전트 협업 프레임워크로는 CAMEL(역할극 기반 자율 협업), AutoGen(프로그래밍 가능한 대화형 에이전트), MetaGPT(SOP 기반 역할 조율) 등이 있으며, 이들은 주로 고정적이거나 밀집(dense)된 통신 패턴에 의존한다. 선택적/동적 통신 토폴로지 연구로는 AgentPrune(메시지 패싱 그래프에서 저가치 메시지 프루닝), G-Designer(태스크 조건부 토폴로지 생성), GTD(가이드 확산 과정을 통한 토폴로지 합성)가 있다. DyTopo는 이들과 달리 에이전트가 자연어 디스크립터를 출력하고 시맨틱 매칭으로 해석 가능한 추론 시점(inference-time) 라우팅을 수행한다는 점에서 차별화된다.

🔬 Main Methodology

DyTopo는 매니저 에이전트가 라운드별 목표를 설정하고, N개의 워커 에이전트가 각 라운드에서 단일 포워드 패스로 공개 메시지, 비공개 메시지, query 디스크립터(현재 필요한 정보), key 디스크립터(제공 가능한 정보)를 동시에 생성하는 구조이다. query와 key 디스크립터는 고정된 사전학습 시맨틱 인코더(Emb)로 벡터화되고, 코사인 유사도 r_{i,j}를 계산한 뒤 하드 임계값 τ_edge를 적용하여 희소 방향 그래프 G(t)를 라운드마다 새로 유도한다. 동기화 장벽(synchronization barrier) 이후에만 메모리가 갱신되며, 비공개 메시지는 활성화된 에지를 따라서만 라우팅되어 수신자의 다음 라운드 컨텍스트에 시맨틱 관련도 순서(σ)로 정렬 결합된다. 그래프가 DAG이면 위상 정렬, 순환이 있으면 제한 입력차수(restricted in-degree) 기반 탐욕적 순환 제거 휴리스틱으로 결정론적 집계 순서를 생성한다. 매니저는 모든 공개 메시지를 집계하여 전역 상태를 파악하고, 내부 평가 함수 Φ가 γ_success 이상이면 종료, 아니면 다음 라운드 목표를 갱신하여 브로드캐스트하는 이중(bi-level) 피드백 루프를 형성한다.

🧪 Evaluation

HumanEval, APPS-Competition(코드 생성), MATH-500, Omni-MATH(수학 추론) 4개 벤치마크에서 MiMo-V2-Flash, GPT-oss-120B, Llama3-8B-Instruct, Qwen3-8B 4개 LLM 백본으로 평가했다. DyTopo는 모든 16개 백본-데이터셋 조합에서 최고 성능을 달성했으며, 가장 강한 베이스라인 대비 평균 +6.2 포인트 향상을 보였다. 특히 Llama3-8B 기반 Omni-MATH에서 14.29% → 30.00%(+110%), Qwen3-8B 기반 HumanEval에서 80.49% → 89.63%(+390% vs 단일 에이전트) 등 소형 모델에서의 개선폭이 두드러졌고, GPT-oss-120B 기반 APPS-Competition에서도 60.55% → 69.66%로 강력한 백본에서도 일관된 이점을 보였다.

💡 Key Intuition & Lesson

핵심 인사이트는 멀티에이전트 통신 토폴로지를 고정 설계가 아니라 라운드별로 재구성되는 적응적 객체로 취급해야 한다는 것이며, 이를 위해 별도의 학습 없이 에이전트 스스로가 자연어로 "무엇이 필요한지/무엇을 제공할 수 있는지"를 선언하게 하고 시맨틱 임베딩 매칭으로 그래프를 유도하는 것만으로도 충분히 효과적이라는 점이다. 또한 이 방식은 정확도 향상 외에도 라운드별 통신 그래프의 변화를 추적할 수 있어 에이전트 간 협업 과정의 해석 가능성(interpretability)을 부수적으로 확보한다는 실용적 이점이 있다.
