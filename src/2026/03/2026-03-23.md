## 📝 상세 분석

### 1. OxyGen: Unified KV Cache Management for Vision-Language-Action Models under Multi-Task Parallelism

**트랙**: ML Systems | **링크**: [arXiv](https://arxiv.org/abs/2603.14371)

도구 권한이 계속 차단되고 있네요. 제공해주신 abstract 정보를 기반으로, 그리고 논문의 기술적 맥락에 대한 제 지식을 결합하여 분석하겠습니다.

📋 Problem Definition

Embodied AI 에이전트는 로봇 조작(manipulation), 대화(conversation), 메모리 구축 등 여러 태스크를 공유된 관측(observation)으로부터 서로 다른 시간 제약 하에 병렬로 수행해야 한다. 최근 Mixture-of-Transformers(MoT) 기반 Vision-Language-Action 모델(VLA)이 이러한 이종(heterogeneous) 출력을 아키텍처적으로 지원하지만, 기존 추론 시스템은 각 태스크가 KV 캐시를 독립적으로 관리하여 중복 연산과 리소스 경쟁이 발생하므로, 온디바이스 멀티태스크 병렬 추론이 비효율적이다. 이 논문은 이 문제의 근본 원인을 "격리된 KV 캐시 관리(isolated KV cache management)"로 진단하고, 이를 해결하는 통합 패러다임을 제안한다.

📚 Background / Related Works

VLA 모델 분야에서는 π₀ 및 π₀.5(Physical Intelligence)가 대표적인 MoT 기반 VLA로, 언어와 액션 출력에 대해 modality별로 분리된 transformer 파라미터를 사용하면서도 비전 입력은 공유하는 구조를 채택하고 있다. 추론 최적화 측면에서는 vLLM, TensorRT-LLM 등 LLM 서빙 시스템이 PagedAttention과 continuous batching을 도입하여 KV 캐시 메모리 효율과 처리량을 높여왔으나, 이들은 단일 모달리티(텍스트) 생성에 초점을 맞추고 있어 VLA의 멀티태스크 병렬 시나리오에는 직접 적용이 어렵다. 또한 로봇 제어 루프에서 액션 생성은 고정 주기(예: 50~70Hz)로 이루어져야 하는 반면 언어 디코딩은 가변 길이로 진행되는 이질적 시간 요구사항이 존재하며, 기존 시스템은 이를 통합적으로 다루지 못했다.

🔬 Main Methodology

OxyGen의 핵심 아이디어는 KV 캐시를 태스크 간에 공유되는 일급(first-class) 자원으로 취급하는 "통합 KV 캐시 관리(unified KV cache management)" 패러다임이다. 이 패러다임은 두 가지 핵심 최적화를 가능하게 한다. 첫째, cross-task KV sharing은 동일한 관측 입력(비전 토큰 등)에 대해 여러 태스크가 각각 prefill을 수행하는 대신, 한 번 계산된 KV 캐시를 공유함으로써 중복 연산을 제거한다. 예를 들어 액션 생성과 언어 응답이 같은 카메라 프레임을 관측하더라도 비전 인코딩 및 해당 KV 캐시 계산은 한 번만 수행된다. 둘째, cross-frame continuous batching은 제어 주기(control cycle)를 넘나들며 고정 주기 액션 생성과 가변 길이 언어 디코딩을 분리(decouple)하여 하나의 배치 안에서 동시에 처리한다. 이는 기존 LLM 서빙의 continuous batching 개념을 시간 축으로 확장한 것으로, 액션 태스크는 매 프레임마다 새로운 요청으로 들어오고 언어 태스크는 여러 프레임에 걸쳐 디코딩이 지속되는 비대칭 구조를 하나의 스케줄러로 통합 관리한다. 이 시스템은 현재 가장 대표적인 MoT VLA인 π₀.5를 대상으로 구현되었다.

🧪 Evaluation

대표적인 로보틱스 구성(representative robotic configurations) 환경에서 평가를 진행했으며, 격리된 실행(isolated execution) 대비 최대 3.7배 속도 향상을 달성했다. 언어 처리량은 200 tokens/s 이상, 액션 생성 주파수는 70Hz를 동시에 달성하면서도 액션 품질(action quality)의 저하가 없음을 확인했다. 이는 실시간 로봇 제어에 필요한 고주파 액션 생성과 사용자 대화를 위한 높은 언어 처리량이라는 두 가지 상충되는 요구사항을 동시에 충족시키는 결과이다.

💡 Key Intuition & Lesson

이 논문의 핵심 인사이트는 멀티모달 멀티태스크 모델에서 추론 병목의 근본 원인이 "모델 아키텍처"가 아니라 "런타임의 자원 관리 추상화 수준"에 있다는 점이다. MoT VLA는 이미 아키텍처적으로 멀티태스크를 지원하지만, 추론 시스템이 각 태스크를 독립적으로 다루면 공유 가능한 연산이 중복되고 이질적 시간 요구사항 간 스케줄링이 비효율적으로 된다. KV 캐시라는 중간 표현을 태스크와 시간 프레임을 가로지르는 공유 자원으로 격상시키는 것만으로 큰 성능 이득을 얻을 수 있으며, 이는 향후 embodied AI 시스템의 온디바이스 배포에서 모델 설계와 시스템 설계를 함께 고려하는 co-design 접근의 중요성을 보여준다.

### 2. DyTopo: Dynamic Topology Routing for Multi-Agent Reasoning via Semantic Matching

**트랙**: Agents | **링크**: [arXiv](https://arxiv.org/abs/2602.06039)

📋 Problem Definition

LLM 기반 멀티에이전트 시스템에서 에이전트 간 통신 구조(topology)가 고정되어 있다는 문제를 다룬다. 멀티라운드 추론은 단계별로 요구사항이 다른데(초기에는 넓은 탐색, 후기에는 정밀한 검증), 기존 시스템은 브로드캐스트나 고정 턴제 같은 정적 토폴로지를 전 라운드에 걸쳐 동일하게 사용하므로, 각 단계의 실제 정보 수요와 통신 패턴 사이에 불일치가 발생한다. DyTopo는 매 라운드마다 에이전트 간 통신 그래프를 동적으로 재구성하여 이 문제를 해결하고자 한다.

📚 Background / Related Works

멀티에이전트 협업 프레임워크로는 역할 기반 대화를 제안한 CAMEL(Li et al., 2023), 프로그래밍 가능한 에이전트 상호작용 패턴을 제공하는 AutoGen(Wu et al., 2024), SOP 기반 조율을 도입한 MetaGPT(Hong et al., 2023) 등이 있으나, 이들은 모두 고정 또는 밀집 통신 패턴에 의존한다. 선택적/동적 토폴로지 방향에서는 AgentPrune(Zhang et al., 2024a)이 시공간 메시지 패싱 그래프에서 저가치 메시지를 가지치기하고, G-Designer(Zhang et al., 2024b)가 태스크 조건부 에이전트 통신 토폴로지를 생성하며, GTD(Jiang et al., 2025)가 가이드 디퓨전 프로세스로 토폴로지를 합성하는 접근을 취했다. DyTopo는 이들과 달리 에이전트가 자연어 기술자(descriptor)를 출력하고 시맨틱 매칭으로 방향 그래프를 구성하는 해석 가능한 추론 시점 라우팅 메커니즘을 제안한다.

🔬 Main Methodology

DyTopo는 Manager 메타에이전트와 N개의 워커 에이전트로 구성된 계층적 프레임워크다. 매 라운드 t에서 각 워커 에이전트는 단일 포워드 패스로 공개 메시지, 비공개 메시지, 그리고 두 개의 경량 자연어 기술자를 생성한다: query 기술자(현재 필요한 정보)와 key 기술자(제공 가능한 정보). 이 기술자들을 사전학습된 시맨틱 인코더로 임베딩한 뒤 코사인 유사도를 계산하고, 하드 임계값 τ_edge를 적용하여 희소 방향 통신 그래프 G(t)를 구성한다. 비공개 메시지는 이 그래프의 활성화된 에지를 따라서만 라우팅되며, 동기화 장벽(synchronization barrier) 이후에 수신자의 메모리에 시맨틱 관련도 순으로 정렬되어 추가된다. Manager는 각 라운드 종료 시 공개 메시지를 집약하여 글로벌 상태를 파악하고, 다음 라운드 목표를 업데이트하거나 종료를 결정하는 이중 수준(bi-level) 피드백 루프를 형성한다. 그래프에 순환이 있을 경우 제한 차수 기반 탐욕적 순환 제거 휴리스틱으로 결정론적 메시지 순서를 보장한다.

🧪 Evaluation

코드 생성(HumanEval, APPS-Competition)과 수학 추론(MATH-500, Omni-MATH) 벤치마크에서 MiMo-V2-Flash, GPT-oss-120B, Llama3-8B-Instruct, Qwen3-8B 네 가지 LLM 백본으로 평가했다. DyTopo는 가장 강한 베이스라인(AgentScope) 대비 평균 +6.2%p 향상을 달성했으며, 특히 Llama3-8B-Instruct에서 Omni-MATH 기준 단일 에이전트 14.29% 대비 30.00%(+110%), Qwen3-8B에서 HumanEval 기준 18.29% 대비 89.63%(+390%)의 극적인 개선을 보였다. 모든 벤치마크-백본 조합 16개 중 DyTopo가 전부 최고 성능을 기록했고, 랜덤 토폴로지나 고정 토폴로지 대비 일관된 우위를 보여 동적 토폴로지의 효과를 입증했다.

💡 Key Intuition & Lesson

핵심 통찰은 멀티에이전트 추론에서 "누가 누구에게 말하는가"라는 통신 구조 자체가 정적 설계 선택이 아니라 매 라운드 적응해야 하는 동적 객체라는 것이다. Transformer의 query-key 어텐션 메커니즘을 에이전트 간 통신 라우팅에 자연어 수준에서 적용한 것이 이 논문의 독창적 발상이며, 에이전트가 스스로 "무엇이 필요하고 무엇을 줄 수 있는지"를 기술하게 함으로써 학습 없이도(training-free) 해석 가능하고 적응적인 통신 토폴로지를 구현할 수 있음을 보여준다. 이는 멀티에이전트 시스템 설계 시 통신 토폴로지를 하이퍼파라미터가 아닌 추론 시점의 일급 객체(first-class object)로 다뤄야 한다는 교훈을 준다.
