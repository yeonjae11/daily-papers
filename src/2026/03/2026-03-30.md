## 📝 상세 분석

### 1. [Pool] 2410.21680

**트랙**: ML Systems | **링크**: [arXiv](https://arxiv.org/abs/2410.21680)

📋 Problem Definition

대규모 AI 모델 학습이 수만 개의 GPU를 동시에 사용하는 규모로 확대되면서, 하드웨어 장애가 "발생할지 여부"가 아니라 "언제 발생하느냐"의 문제가 되었다. 이 논문은 Meta의 24k NVIDIA A100 GPU로 구성된 두 개의 연구 클러스터(RSC-1: 16k GPU, RSC-2: 8k GPU)에서 11개월간 수집한 실제 운영 데이터를 바탕으로, 대규모 ML 학습 인프라의 장애 특성을 체계적으로 분석하고 신뢰성 향상 방안을 제시한다. 특히 기존 연구가 LLM 전용 클러스터에 집중한 것과 달리, 1-GPU부터 4k+ GPU까지 다양한 연구 워크로드를 동시에 서비스하는 범용 연구 클러스터의 인프라 과제를 다룬다.

📚 Background / Related Works

대규모 학습 인프라 경험을 공유한 선행 연구로는 ByteDance의 MegaScale, Google의 Gemini 학습 인프라 보고서, 그리고 Meta 자체의 LLaMa 학습 경험이 있으며, 이들은 각각 특정 모델 학습에 초점을 맞추었다. GPU 클러스터의 하드웨어 신뢰성 분석으로는 대규모 GPU 장애율 연구들이 존재하지만, 10^5 GPU 규모의 ML 연구 워크로드 전반에 대한 체계적 인프라 분석은 이 논문이 최초라고 주장한다. 체크포인팅 최적화와 관련해서는 MTTF 기반 최적 체크포인트 주기 산출 연구(Daly 등)가 참조되었고, 클러스터 효율성 메트릭으로는 Google Cloud의 "Runtime Goodput" 개념과 기존의 job slowdown 메트릭이 비교 대상으로 언급된다.

🔬 Main Methodology

첫째, 장애 분류 체계(failure taxonomy)를 제안한다. 증상(GPU 접근 불가, NCCL 타임아웃, ECC 오류 등)과 장애 도메인(사용자 코드, 시스템 소프트웨어, 하드웨어)을 교차 매핑하여, 하나의 증상이 여러 원인에 기인할 수 있음을 체계화하고 "감별 진단(differential diagnosis)" 접근법을 제안한다. 둘째, Effective Training Time Ratio(ETTR)라는 핵심 메트릭을 정의하여, 체크포인트 복구 시간, 재시작 오버헤드, 체크포인트 저장 오버헤드를 모두 고려한 실질 학습 효율을 측정한다. ETTR은 생산적 런타임을 가용 벽시계 시간(대기열 시간 포함)으로 나눈 비율로, 0에서 1 사이의 값을 가진다. 셋째, GPU 규모별 MTTF를 분석적으로 추정하는 방법과, 다양한 작업 파라미터를 함수로 하는 예상 ETTR 추정기를 설계한다. 넷째, 5분 주기의 노드 헬스체크 시스템을 운영하여 "한 번 장애가 난 노드에서 두 번째 작업 실패가 발생하지 않도록" 결함 노드를 자동으로 격리하고, 적응형 라우팅 등 소프트웨어 완화책을 적용한다.

🧪 Evaluation

11개월간 두 클러스터에서 약 400만 개의 작업, 1억 5천만 A100 GPU 시간 이상의 데이터를 분석했다. RSC-1은 일평균 7.2k개, RSC-2는 4.4k개의 작업이 제출되며, 각각 83%와 85%의 클러스터 활용률을 기록했다. 4k GPU 이상을 사용하는 대규모 작업은 전체 작업의 1% 미만이지만 GPU 자원의 12%를 소비하는 것으로 나타났고, 헬스체크의 오탐률(false positive rate)은 성공적으로 완료된 작업 중 1% 미만으로 보정되었다. PCIe 장애가 GPU XID 이벤트 없이도 GPU 접근 불가를 유발하는 경우가 RSC-1에서 57%, RSC-2에서 37% 관찰되어, 중첩적 헬스체크의 필요성이 실증적으로 확인되었다. MTTF 예측은 이론적 모델과 일치했으며, 4k GPU 규모까지의 실제 작업 데이터로 검증되었다.

💡 Key Intuition & Lesson

가장 핵심적인 교훈은 "하나의 불량 노드가 전체를 망친다(One bad node spoils the bunch)"는 원칙으로, gang scheduling 환경에서는 단 하나의 노드 장애가 수천 개 GPU를 유휴 상태로 만들기 때문에, 결함 노드를 신속하게 탐지하고 격리하는 헬스체크 시스템이 대규모 학습의 전제 조건이다. NCCL 타임아웃처럼 네트워크 문제로 보이는 증상이 실제로는 데이터 로딩 지연이나 사용자 코드 버그에서 기인하는 "레드 헤링" 현상이 빈번하므로, 단일 증상에 원인을 성급히 귀속하지 않는 감별 진단 접근이 필수적이다. 또한 LLM 전용이 아닌 범용 연구 클러스터에서는 워크로드 다양성이 훨씬 크기 때문에, 특정 모델에 최적화된 인프라보다 워크로드에 무관한(workload-agnostic) 범용 신뢰성 기법이 더 효과적이라는 점이 실데이터로 입증되었다.

### 2. [Pool] 2503.06072

**트랙**: LLM Post-training | **링크**: [arXiv](https://arxiv.org/abs/2503.06072)

📋 Problem Definition

이 논문은 LLM의 사후 훈련(Post-training) 기법들이 ChatGPT의 RLHF부터 DeepSeek-R1의 cold-start RL에 이르기까지 급격히 발전해왔음에도, 이를 체계적으로 종합·분류한 서베이가 부재하다는 문제를 다룬다. 기존 서베이들은 선호도 정렬, 파라미터 효율적 파인튜닝 등 개별 하위 주제에만 집중했으며, 파인튜닝·정렬·추론·효율성·통합/적응이라는 다섯 축을 아우르는 전체론적 조망이 없었다. 이에 Post-training Language Models(PoLMs)라는 개념을 정립하고, 사전 훈련된 모델을 특정 태스크와 사용자 요구에 맞추는 모든 기법을 하나의 프레임워크 안에서 정리하고자 한다.

📚 Background / Related Works

사전 훈련 패러다임의 기원으로 BERT(2018)와 GPT(2018)가 "pre-training and fine-tuning" 학습 패러다임을 확립했으며, T5(2019)가 텍스트-투-텍스트 프레임워크로 다중 태스크 학습의 기반을 놓았다. 사후 훈련 측면에서는 InstructGPT의 RLHF, PPO 기반 정렬 안정화, DPO(Rafailov et al., 2023)의 보상 모델 없는 직접 선호도 최적화, Chain-of-Thought 프롬프팅(Wei et al.), 그리고 RAG, LoRA 등 파라미터 효율적 기법들이 핵심 선행 연구로 다루어진다. 특히 DeepSeekMath에서 처음 제안된 GRPO(Group Relative Policy Optimization)가 PPO의 critic 모델을 그룹 스코어 기반 베이스라인으로 대체하여 훈련 비용을 줄인 점, 그리고 Switch Transformer(1.6조 파라미터, 2048 expert)와 Mixtral로 대표되는 MoE 아키텍처의 발전이 효율성 관련 주요 선행 작업으로 정리된다.

🔬 Main Methodology

이 서베이의 핵심 기여는 PoLMs를 다섯 가지 범주로 분류하는 구조화된 택소노미를 제안한 것이다. (1) 파인튜닝: SFT(Supervised Fine-Tuning)와 RFT(Reinforcement Fine-Tuning)로 나누어 레이블된 데이터 기반 파라미터 조정 기법을 정리한다. (2) 정렬(Alignment): RLHF, RLAIF(AI 피드백 기반 강화학습), DPO의 세 갈래로 분류하며, PPO의 클리핑 목적함수(식 2)를 통한 안정적 정책 업데이트와 DPO의 Bradley-Terry 모델 기반 쌍대 비교 선호도 최적화(식 7)를 수학적으로 정식화한다. (3) 추론(Reasoning): Self-Refinement과 RL for Reasoning으로 나누며, DeepSeek-R1-Zero의 자기 검증·반성·확장된 CoT 생성이 SFT 없이도 RL만으로 추론 능력을 발현시킨 사례를 중점적으로 다룬다. (4) 효율성: 모델 압축, PEFT, 지식 증류를 포괄하며, MoE 아키텍처(DeepSeek V2.5의 총 2360억 파라미터 중 210억 활성, 160 expert)의 효율적 파라미터 활용을 분석한다. (5) 통합 및 적응: 멀티모달 접근(PaLM-E, LLaVA 등), 도메인 적응(RAG), 모델 병합 기법을 다룬다. 또한 GRPO의 목적함수(식 8)를 통해 critic 모델 없이 그룹 내 출력 샘플들의 상대적 점수로 advantage를 추정하는 방식을 수학적으로 제시한 점이 방법론적으로 중요하다.

🧪 Evaluation

이 논문은 실험 논문이 아닌 서베이 논문이므로 자체적인 벤치마크 실험 결과를 포함하지 않는다. 대신, 서베이 대상 모델들의 성과를 정리하는데, 예를 들어 DeepSeek V2.5는 2360억 총 파라미터에서 210억만 활성화하는 MoE 구조로 LMSYS 벤치마크에서 state-of-the-art를 달성했으며, Switch-C Transformer는 1.6조 파라미터를 2048개 expert에 분산시켜 dense 모델 대비 동등한 성능을 훨씬 적은 연산으로 구현했음을 보고한다. 데이터셋은 7가지 유형으로 분류하여 PromptSource, Flan, CNN/DailyMail, TL;DR 등 주요 사후 훈련 데이터셋의 특성과 용도를 체계적으로 정리하였다.

💡 Key Intuition & Lesson

이 서베이에서 얻을 수 있는 가장 중요한 인사이트는 사후 훈련의 패러다임이 "인간 피드백 기반 정렬"에서 "RL 기반 자율적 추론 능력 발현"으로 전환되고 있다는 점이다. DeepSeek-R1-Zero가 SFT 없이 순수 RL만으로 자기 검증과 반성 능력을 획득한 사례는, 사전 훈련에서 축적된 지식이 적절한 RL 인센티브만으로도 고차원 추론으로 발현될 수 있음을 보여준다. 또한 GRPO처럼 critic 모델을 제거하고 그룹 상대 스코어로 대체하는 접근은, 사후 훈련의 연산 비용을 대폭 줄이면서도 성능을 유지할 수 있는 실용적 방향을 제시하며, 향후 PoLMs 연구가 추론 능력 확장과 훈련 효율성이라는 두 축을 동시에 추구해야 함을 시사한다.
