## 📝 상세 분석

### 1. Improving training time and GPU utilization in geo-distributed language model training

**트랙**: ML Systems | **링크**: [arXiv](https://arxiv.org/abs/2411.14458)

📋 Problem Definition

대규모 언어모델(LM) 훈련에는 수만 개의 GPU가 필요하지만, 전력 밀도, 냉각, 공간 제약 등으로 인해 이를 단일 데이터센터(DC)에 집중 배치하기가 점점 어려워지고 있다. 이 논문은 WAN(광역 네트워크)으로 연결된 여러 데이터센터에 분산된 GPU를 활용하여 LM을 훈련할 때 발생하는 통신 병목과 GPU 유휴 시간(bubble) 문제를 해결하고자 한다. 기존 방식으로는 WAN의 낮은 대역폭과 높은 지연시간 때문에 단일 DC 대비 훈련 시간이 15배 이상 느려지고 GPU 활용률이 5% 미만으로 떨어지는 심각한 성능 저하가 발생한다.

📚 Background / Related Works

기존 분산 훈련 연구인 GPipe, PipeDream, Megatron-LM, Varuna 등은 모든 GPU가 단일 DC 내에 있다고 가정하고 파이프라인 병렬성(PP) 스케줄링을 최적화했으나, WAN 환경의 저대역폭 문제를 고려하지 않았다. 지리 분산 훈련을 다룬 GAIA는 Virginia-Sao Paulo 간 100Mbps 수준의 WAN 대역폭만 활용 가능했고, Decentralized Training 등도 WAN 대역폭의 제약을 극복하지 못했다. 추론 측면에서는 Splitwise와 DistServe가 prefill과 decode 단계를 분리하는 아키텍처를 제안했는데, BubbleTea는 이 아이디어를 훈련 중 유휴 시간 활용에 접목하였다.

🔬 Main Methodology

Atlas는 세 가지 핵심 설계를 통해 지리 분산 훈련 시간을 단축한다. 첫째, PyTorch가 WAN에서 노드 간 단일 TCP 연결만 사용하여 대역폭이 수백 Mbps에 그치는 문제를, 다중 TCP 연결을 병렬로 생성하여 거리에 관계없이 5Gbps까지 대역폭을 확장하는 방식으로 해결한다. 둘째, 데이터 병렬성(DP)의 여러 파이프라인 간 WAN 대역폭을 시간적으로 지능적으로 공유(workload-aware temporal bandwidth sharing)하여, 후단 파이프라인 스테이지의 마이크로배치 처리를 앞당기고 마이크로배치 간 bubble을 제거한다. 셋째, DC 간 GPU 수의 불균형이 심할 경우(예: 1000 대 10) 오히려 소수 DC의 GPU를 사용하지 않는 것이 유리할 수 있으므로, 최적의 DC별 GPU 분배를 계산하는 휴리스틱을 제공한다. BubbleTea는 Atlas가 제거하지 못하는 forward-backward pass 사이의 잔여 bubble에 LM 추론의 prefill 단계를 스케줄링하는 prefill-as-a-service를 구축하여, 훈련에 영향을 주지 않으면서 유휴 GPU 사이클을 활용한다. Prefill의 완료 시간이 입력 프롬프트 길이에 의해 결정론적이므로 bubble 내 스케줄링이 정확하게 가능하다는 점이 핵심이다.

🧪 Evaluation

3개 DC에 걸쳐 6개 A100 GPU를 배치한 테스트베드와 대규모 시뮬레이션을 통해 평가했으며, GPT-3급(GPT-A: L=4K, H=4K, 레이어당 4.12억 파라미터)과 더 큰 모델(GPT-B: L=6K, H=8K, 레이어당 12억 파라미터)을 사용했다. Atlas는 Varuna 대비 최대 17배 빠른 훈련 시간을 달성했고, 대역폭 공유 스케줄링만으로도 기존 최고 스케줄러 대비 1.82배 개선을 보였다. BubbleTea와 결합 시 GPU 활용률이 최대 94%에 도달했으며, 이는 기존 지리 분산 방식의 5% 미만 활용률과 극명한 대비를 이룬다.

💡 Key Intuition & Lesson

가장 큰 교훈은 "단순한 아이디어가 큰 효과를 낼 수 있다"는 것이다. PyTorch가 WAN에서 단일 TCP 연결만 사용한다는 사실을 발견하고, 다중 TCP 연결로 대역폭을 20배 확장한 것이 전체 개선의 상당 부분을 차지하며, 이는 시스템 수준의 세밀한 관찰이 알고리즘적 혁신 못지않게 중요함을 보여준다. 또한 훈련과 추론을 별개의 워크로드로 보지 않고, 훈련 중 불가피한 유휴 시간에 추론의 prefill을 끼워 넣는 co-location 전략은 GPU 자원의 경제성을 근본적으로 재고하게 만드는 관점의 전환이다.

### 2. From System 1 to System 2: A Survey of Reasoning Large Language Models

**트랙**: LLM Post-training | **링크**: [arXiv](https://arxiv.org/abs/2502.17419)

📋 Problem Definition

현재의 기초 LLM(Foundational LLM)은 빠른 직관적 판단(System 1)에는 뛰어나지만, 복잡한 문제에 대한 단계적이고 논리적인 심층 추론(System 2)에는 한계를 보인다. 이 서베이는 System 1에서 System 2로의 전환을 어떻게 달성할 수 있는지, 즉 LLM이 인간 수준의 숙고적 추론(deliberate reasoning) 능력을 갖추기 위한 방법론과 발전 경로를 체계적으로 정리하는 것을 목표로 한다. 특히 OpenAI o1/o3, DeepSeek R1 등 최근 등장한 추론 특화 LLM들이 수학, 코딩 등에서 전문가 수준 성능을 보이는 현상을 포괄적으로 분석하고자 한다.

📚 Background / Related Works

이 서베이는 Daniel Kahneman의 이중 처리 이론(Dual Process Theory)에서 정의한 System 1(빠르고 직관적)과 System 2(느리고 숙고적) 사고 체계를 LLM 맥락으로 가져온다. 선행 작업으로는 Chain-of-Thought(CoT) 프롬프팅(Wei et al., 2022), Tree-of-Thought(Yao et al., 2023), Self-Consistency(Wang et al., 2023) 등 초기 System 2 기술들이 기초 LLM의 추론 능력을 향상시키려는 시도로 다뤄진다. 또한 OpenAI의 o1/o3 시리즈, DeepSeek의 R1, Google의 Gemini 2.0 Flash Thinking 등 최신 추론 LLM들의 발전 계보를 추적하며, RLHF 및 강화학습 기반 후훈련(post-training) 방법론의 진화를 함께 조망한다.

🔬 Main Methodology

이 서베이의 핵심 기여는 추론 LLM을 구축하기 위한 방법론을 체계적으로 분류하고 분석한 것이다. 먼저 기초 LLM의 발전 과정(GPT 시리즈, LLaMA, Gemini 등)을 개관한 후, System 2 추론을 가능하게 하는 핵심 기법들을 정리한다. 여기에는 (1) 추론 시점의 스케일링(test-time compute scaling)으로 추론 단계에서 더 많은 연산을 투입하는 방법, (2) 강화학습 기반 훈련으로 모델이 스스로 추론 체인을 생성하고 검증하도록 학습시키는 방법, (3) 지도학습 기반 증류(distillation)로 대형 추론 모델의 능력을 소형 모델에 전이하는 방법이 포함된다. 또한 다양한 추론 LLM들의 특징을 비교 분석하고, 각 모델이 어떤 기법 조합으로 고급 추론 능력을 달성했는지를 계보적으로 추적한다. GitHub 리포지토리(Awesome-Slow-Reason-System)를 통해 실시간으로 최신 동향을 업데이트하는 살아있는 서베이(living survey) 형태를 취한다.

🧪 Evaluation

서베이에서는 수학(MATH, GSM8K, AMC, AIME 2024), 코딩(Codeforces, LiveCodeBench, SWE-bench Verified), 과학(GPQA Diamond) 등 주요 추론 벤치마크에서의 대표 모델 성능을 비교한다. 예를 들어 OpenAI o1은 AIME 2024에서 83.3%, GPQA Diamond에서 78.0%, Codeforces에서 89 퍼센타일을 달성했으며, DeepSeek R1은 AIME 2024에서 79.8%, MATH-500에서 97.3%를 기록해 오픈소스 모델로서 상용 모델에 근접한 성능을 보였다. 이러한 결과는 추론 특화 LLM이 기존 기초 LLM 대비 복잡한 문제 해결에서 현격한 성능 향상을 달성했음을 실증적으로 보여준다.

💡 Key Intuition & Lesson

핵심 인사이트는 LLM의 추론 능력 향상이 단순히 모델 크기를 키우는 것(training-time scaling)이 아니라, 추론 시점에서의 연산 확장(test-time compute scaling)과 강화학습을 통한 자기 개선(self-improvement) 메커니즘의 결합에서 비롯된다는 점이다. 또한 DeepSeek R1의 사례가 보여주듯, 대규모 추론 모델의 능력을 증류를 통해 소형 모델로 효과적으로 전이할 수 있어, 추론 능력의 민주화 가능성이 열리고 있다. 이 서베이는 System 2 추론이 단일 기술이 아닌 여러 기법의 상호보완적 조합으로 달성되며, 향후 멀티모달 추론, 에이전트 기반 추론 등으로 확장될 수 있는 방향성을 제시한다.
